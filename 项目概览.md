# 项目概览

## 项目名称

在线教学教务管理平台（EduAdmin）

## 项目目标

围绕教培业务搭建一个可联调、可部署、可持续扩展的教务系统，打通招生建档、排课、上课、录屏、扣课时、收款等核心流程。

## 当前架构

| 层级 | 技术选型 |
|------|----------|
| 管理后台 | Vue3 + Element Plus + Vite |
| 课堂端 | Taro（React） |
| 后端 | Django 4.2 + DRF |
| 数据库 | PostgreSQL（目标）/ SQLite（当前默认开发） |
| 缓存与异步 | Redis / Celery |
| 实时课堂 | 腾讯会议 SDK（当前以 Mock 为主） |

## 业务模块

| 模块 | 说明 |
|------|------|
| 用户权限 | 登录、认证、角色、权限、菜单 |
| 教务管理 | 学生、教师、课程、班级、排课、课时 |
| 在线课堂 | 会议、录制、回放、课堂备注 |
| 考试管理 | 题目、试卷、考试、成绩 |
| 财务管理 | 订单、支付、退款、统计 |

## 当前评估

- Django 后端与管理后台的结构完整度较高，已经具备继续联调的基础。
- 多端课堂应用已有框架和部分页面，但真实功能完成度仍需逐页验证。
- 数据库设计文档较完整，但运行配置仍需从 SQLite 向 PostgreSQL 目标形态收敛。
- 测试、运维、用户手册目录已建立，但部分内容仍以模板和初稿为主。

## 当前重点问题

1. 文档中原有的完成度描述存在夸大和不同步问题，已开始清理。
2. Docker 目标环境与 Django 默认开发配置不完全一致。
3. 腾讯会议真实账号未接入，课堂链路仍主要依赖 Mock。
4. 测试和验收结果文档需要补齐，不能只保留目录结构。
