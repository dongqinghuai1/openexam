# 项目概览

## 项目定位

OPENEXAM 是一个多角色教育业务平台示例工程，目标是把用户权限、教务流程、课堂联动、考试流程和财务流程串成一套可运行、可联调、可继续扩展的系统。

当前仓库不是单一前端项目，也不是单一后端项目，而是一个包含产品文档、前端工程、后端工程、数据库设计、测试验收和运维文档的完整工作区。

## 技术架构

| 层级 | 技术选型 |
|------|----------|
| 管理后台 | Vue 3 + Vite + Element Plus |
| 教师 / 学生 / 家长端 | Taro + React |
| 后端 | Django 4.2 + DRF |
| 数据库 | PostgreSQL 或 SQLite |
| 缓存 | Redis 或本地内存缓存 |
| 部署 | Docker Compose |

## 当前后端模块

后端主工程已注册并对外暴露以下 API 模块：

- `users`：用户、登录、角色、菜单、通知、操作日志、Dashboard、邮箱验证码注册与重置密码
- `edu`：科目、课程、学生、教师、班级、排课、调课、请假、课时账户、课时流水
- `classroom`：会议室、课堂记录、录制任务、回放文件、课堂笔记
- `exam`：题目、试卷、考试、成绩
- `finance`：订单、退款、退款申请与统计

## 当前成熟度判断

### 已经比较明确的部分

- 后端主模块和 API 路由已经落地。
- 管理后台与后端、数据库、Redis 的容器化链路已经具备。
- 教师端、学生端、家长端已支持 H5 构建和 Docker 部署。
- 登录页已支持邮箱注册、忘记密码和按角色统一分流。

### 仍需保守描述的部分

- 教师端、学生端、家长端当前更适合演示和业务联调，不应直接等同于生产上线形态。
- 课堂能力已有状态流转和回放链路，但目前主要用于演示与联调。
- 生产级部署、安全、监控和第三方接入仍需继续完善。

## 当前登录策略

- 管理后台只允许管理员进入。
- 教师、学生、家长可在管理后台登录页完成统一登录入口分流。
- 非管理员登录后会自动跳转到对应端，并自动带上登录信息。

## 一句话总结

如果你现在要接手这个仓库，最稳妥的理解是：这是一个已经具备“四端前端 + Django 后端 + PostgreSQL + Redis + 邮箱注册分流”的教育业务平台工作区，当前可以完整跑通多角色登录与主要业务链路，但仍不是面向生产环境的最终形态。
