# 项目概览

## 项目定位

OPENEXAM 是一个多角色教育业务平台示例工程，目标是把用户权限、教务流程、课堂联动、考试流程和财务流程串成一套可运行、可联调、可继续扩展的系统。

当前仓库不是单一前端项目，也不是单一后端项目，而是一个包含产品文档、前端工程、后端工程、数据库设计、测试验收和运维文档的完整工作区。

## 技术架构

| 层级 | 技术选型 |
|------|----------|
| 管理后台 | Vue 3 + Vite + Element Plus |
| 教师 / 学生 / 家长端 | Taro + React |
| 后端 | Django 4.2 + DRF |
| 数据库 | PostgreSQL 或 SQLite |
| 缓存 | Redis 或本地内存缓存 |
| 部署 | Docker Compose |

## 当前后端模块

后端主工程已注册并对外暴露以下 API 模块：

- `users`：用户、登录、角色、菜单、通知、操作日志、Dashboard
- `edu`：科目、课程、学生、教师、班级、排课、调课、请假、课时账户、课时流水
- `classroom`：会议室、课堂记录、录制任务、回放文件、课堂笔记
- `exam`：题目、试卷、考试、成绩
- `finance`：订单、退款、退款申请与统计

## 当前可运行入口

### 默认 Docker 部署

默认会启动：

- PostgreSQL
- Redis
- Django 后端
- Vue 管理后台

### 源码级入口

- 管理后台可本地 `npm run dev`
- 教师端、学生端、家长端可本地 `npm run dev:h5`
- 后端可本地 `python manage.py runserver 8000`

## 当前成熟度判断

### 已经比较明确的部分

- 后端主模块和 API 路由已经落地。
- 管理后台与后端、数据库、Redis 的容器化链路已经具备。
- 初始化命令已补齐，首次启动可自动生成科目和演示账号。
- 启动验证报告已经记录了后端检查、管理后台构建和容器代理修复过程。

### 仍需保守描述的部分

- 教师端、学生端、家长端存在独立工程，但当前应视为演示版客户端。
- 课堂能力已有后端接口和状态流转，但目前主要用于 Mock/演示链路。
- 生产级部署、监控、安全和真实第三方接入仍需继续完善。
