## 后端接口核验结果

### 用户模块 `users`

已实现：

- 登录 `POST /users/login/`
- 用户 CRUD `GET/POST /users/`、`GET/PUT/DELETE /users/{id}/`
- 刷新 Token `POST /users/{router action refresh}`
- 当前用户 `GET /users/{router action me}`
- 角色、菜单、权限接口

发现问题：

- 文档写的是 `POST /users/refresh/`、`GET /users/me/`，而当前代码把这些动作定义在 `UserViewSet` 上，实际路由样式需要以 DRF router 生成结果为准。
- 原先代码中存在未使用的 `rest_framework_simplejwt` 引用，已移除，避免未安装依赖导致启动失败。

### 教务模块 `edu`

已实现：

- 科目、课程、学生、教师、班级、排课、调课、请假、课时账户、课时流水基础 `ViewSet`
- 学生课时账户 `students/{id}/hours_accounts/`
- 学生课表 `students/{id}/schedules/`
- 教师课表 `teachers/{id}/schedules/`
- 班级加人 `classes/{id}/add_student/`
- 班级移除学生 `classes/{id}/remove_student/{student_id}/`
- 排课日历 `schedules/calendar/`
- 课时按学生查询 `hours/flows/by_student/`

发现问题：

- `EduClassViewSet.add_student` 中缺失的 `date` 导入已补齐。

### 课堂模块 `classroom`

已实现：

- 会议室、参会记录、录制任务、回放文件、课堂备注 `ViewSet`
- 创建会议 `meeting_rooms/create_meeting/`
- 获取 Token `meeting_rooms/{id}/get_token/`
- 开始会议 `meeting_rooms/{id}/start_meeting/`
- 结束会议 `meeting_rooms/{id}/end_meeting/`
- 获取回放地址 `playbacks/{id}/url/`

发现问题：

- 腾讯会议服务当前是明确的 Mock 实现，不是真实 SDK 对接。
- `end_meeting` 只会把状态为 `recording` 的录制任务改为 `ready`，但 `start_meeting` 创建任务时状态是 `pending`，状态流转可能不一致。

- 发布考试 `exams/{id}/publish/`

发现问题：
