### 2.2 管理后台容器运行时问题修复

问题现象：

- 浏览器访问 `http://127.0.0.1:3000/` 后，登录请求 `/api/users/login` 返回 `404`。
- `favicon.ico` 返回 `404`。

原因定位：

- 管理后台前端请求使用相对路径 `/api`。
- Vite 开发环境可通过 dev server proxy 转发到 Django。
- 生产容器中使用 `nginx` 托管静态文件，但默认配置未代理 `/api`，因此请求被前端容器本身以静态资源方式处理并返回 `404`。

修复措施：

- 为前端容器增加自定义 `nginx.conf`。
- 将 `/api/` 代理到 `http://backend:8000/api/`。
- 对 `/favicon.ico` 返回 `204`，消除无意义 404 噪音。

验证结果：

- `curl http://127.0.0.1:3000/favicon.ico` 返回 `204`。
- `curl http://127.0.0.1:3000/api/users/` 返回后端 `403` 鉴权响应，说明前端容器到后端的 API 代理已生效。
- 浏览器控制台中的 `Unchecked runtime.lastError: Could not establish connection` 属于浏览器扩展注入噪音，不是项目代码错误。
