# 部署指南

这份文档只描述当前仓库里已经明确存在并且已验证过的部署路径，不把尚未纳入 Docker 编排的客户端工程混写进默认部署范围。

## 1. 当前已支持的部署范围

根目录 `docker-compose.yml` 当前会启动以下服务：

- `eduadmin-postgres`
- `eduadmin-redis`
- `eduadmin-backend`
- `eduadmin-frontend`

对应关系：

- `eduadmin-postgres`：PostgreSQL 14
- `eduadmin-redis`：Redis 7
- `eduadmin-backend`：Django + DRF + Gunicorn
- `eduadmin-frontend`：Vue 3 管理后台静态站点，由 Nginx 提供服务

说明：

- 当前 Docker Compose 不包含教师端、学生端、家长端。
- 三端客户端如需运行，请按各自工程独立启动或单独补充部署方案。

## 2. 环境要求

- Docker Desktop 或可用的 Docker Engine
- Docker Compose
- 能正常拉取基础镜像

如果镜像拉取失败，优先检查网络、镜像源配置或公司代理环境。

## 3. 启动部署

在项目根目录执行：

```bash
docker-compose up -d
```

查看服务状态：

```bash
docker-compose ps
```

查看日志：

```bash
docker-compose logs -f backend
docker-compose logs -f frontend
```

如果你修改了管理后台或后端代码，需要重建镜像：

```bash
docker-compose up -d --build frontend backend
```

## 4. 容器启动时会发生什么

后端镜像启动命令会自动执行：

```bash
python manage.py migrate --noinput
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
gunicorn eduadmin.wsgi:application --bind 0.0.0.0:8000
```

这意味着首次启动后通常会自动完成：

- 数据库迁移
- 默认科目初始化
- 管理员初始化
- 教师、学生、家长演示账号初始化

## 5. 默认访问地址

- 管理后台：`http://127.0.0.1:3000`
- 后端 API：`http://127.0.0.1:8000`
- PostgreSQL：`127.0.0.1:5432`
- Redis：`127.0.0.1:6379`

## 6. 默认账号

- 管理后台：`admin / admin123`
- 教师端：`teacher / teacher123`
- 学生端：`13800000002 / student123`
- 家长端：`13800000003 / parent123`

## 7. 本地非 Docker 运行

### 后端

在 `04_后端开发/03_业务代码` 目录执行：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
python manage.py runserver 8000
```

默认规则：

- 未配置数据库环境变量时，后端使用 SQLite。
- 配置 `DB_ENGINE=postgres` 或 `DB_HOST`、`DB_NAME` 后，后端切换到 PostgreSQL。
- 未配置 `REDIS_URL` 时，缓存退回本地内存缓存。

### 管理后台

在 `03_前端开发/01_管理后台` 目录执行：

```bash
npm install
npm run dev
```

### 三端客户端

教师端、学生端、家长端均为独立 Taro 工程，可分别执行：

```bash
npm install
npm run dev:h5
```

## 8. 部署边界与风险提示

- 当前最稳定的部署组合是“管理后台 + Django 后端 + PostgreSQL + Redis”。
- 三端客户端不在默认容器部署范围，若需要统一发布，需要额外补充网关、构建产物托管与环境变量管理方案。
- 课堂相关能力当前偏演示性质，不建议在文档中将其描述为已完成生产级会议系统接入。
- 当前配置中 `ALLOWED_HOSTS = ['*']`、`CORS_ALLOW_ALL_ORIGINS = True` 更适合开发和演示环境，生产部署前必须收敛。
- 默认账号和默认密码仅适用于本地、演示或测试环境。
