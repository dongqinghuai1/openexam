# 部署指南

这份文档描述的是当前仓库已经落地并且可直接复现的部署方式。当前默认部署不再只有管理后台，而是已经扩展为管理后台、教师端、学生端、家长端四个前端入口共享同一个 Django 后端。

## 1. 当前已支持的部署范围

根目录 `docker-compose.yml` 当前会启动以下服务：

- `eduadmin-postgres`
- `eduadmin-redis`
- `eduadmin-backend`
- `eduadmin-frontend`
- `eduadmin-teacher-portal`
- `eduadmin-student-portal`
- `eduadmin-parent-portal`

对应关系如下：

| 服务名 | 作用 | 默认端口 |
|------|------|----------|
| `eduadmin-postgres` | PostgreSQL 数据库 | `5432` |
| `eduadmin-redis` | Redis 缓存 | `6379` |
| `eduadmin-backend` | Django + DRF + Gunicorn | `8000` |
| `eduadmin-frontend` | Vue 3 管理后台 | `3000` |
| `eduadmin-teacher-portal` | 教师端 H5 | `3001` |
| `eduadmin-student-portal` | 学生端 H5 | `3002` |
| `eduadmin-parent-portal` | 家长端 H5 | `3003` |

## 2. 环境要求

- Docker Desktop 或可用的 Docker Engine
- Docker Compose
- 能正常拉取基础镜像

如果镜像拉取失败，优先检查网络、镜像源配置或代理环境。

## 3. 启动部署

在项目根目录执行：

```bash
docker-compose up -d
```

查看状态：

```bash
docker-compose ps
```

查看日志：

```bash
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f teacher-portal
docker-compose logs -f student-portal
docker-compose logs -f parent-portal
```

如果修改了前后端代码，需要重建镜像：

```bash
docker-compose up -d --build frontend backend teacher-portal student-portal parent-portal
```

## 4. 容器启动时会发生什么

后端镜像启动命令会自动执行：

```bash
python manage.py migrate --noinput
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
gunicorn eduadmin.wsgi:application --bind 0.0.0.0:8000
```

这意味着首次启动通常会自动完成：

- 数据库迁移
- 默认科目初始化
- 默认管理员初始化
- 教师、学生、家长演示账号初始化

## 5. 默认访问地址

- 管理后台：`http://127.0.0.1:3000`
- 教师端：`http://127.0.0.1:3001`
- 学生端：`http://127.0.0.1:3002`
- 家长端：`http://127.0.0.1:3003`
- 后端 API：`http://127.0.0.1:8000`
- PostgreSQL：`127.0.0.1:5432`
- Redis：`127.0.0.1:6379`

- 管理后台：`admin / admin123`
- 教师端：`teacher / teacher123`
- 学生端：`13800000002 / student123`
- 家长端：`13800000003 / parent123`

## 7. 登录与统一分流

当前登录策略如下：

- 管理后台登录页支持：
  - 用户注册
  - 忘记密码
  - 邮箱验证码
- 管理后台仅允许管理员留在后台。
- 非管理员登录管理后台时，会按角色自动跳转到：
  - 教师端 `3001`
  - 学生端 `3002`
  - 家长端 `3003`
- 三端已支持接收统一登录参数并自动落库。

## 8. 邮箱验证码配置

如需启用注册与忘记密码，需要在根目录 `.env` 配置 SMTP 环境变量，例如：

```bash
EMAIL_HOST=smtp.qq.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@example.com
EMAIL_HOST_PASSWORD=your-app-password
EMAIL_USE_TLS=True
DEFAULT_FROM_EMAIL=your-email@example.com
```

说明：

- 学生、家长支持自助注册后直接启用。
- 教师支持自助注册，但默认注册后为待审核状态，需要后台启用后再登录。
- `.env` 已被仓库忽略，适合本地或测试环境保存敏感配置。

## 9. 本地非 Docker 运行

### 后端

在 `04_后端开发/03_业务代码` 目录执行：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
python manage.py runserver 8000
```

### 管理后台

在 `03_前端开发/01_管理后台` 目录执行：

```bash
npm install
npm run dev
```

### 三端客户端

三端本地 H5 默认开发端口：

- 教师端：`3001`
- 学生端：`3002`
- 家长端：`3003`

分别执行：

```bash
npm install
npm run dev:h5
```

## 10. 当前边界与风险提示

- 当前最稳定的部署组合已经是“四端前端 + Django 后端 + PostgreSQL + Redis”。
- 四端前端当前更适合演示、联调和内部验证，不建议直接当成生产终态。
- 当前配置中 `ALLOWED_HOSTS = ['*']`、`CORS_ALLOW_ALL_ORIGINS = True` 更适合开发和演示环境，生产部署前必须收敛。
- 默认账号和默认密码仅适用于本地、演示或测试环境。
- 课堂能力当前可用作演示与联调，但不建议在文档中描述为已完成生产级课堂平台接入。
