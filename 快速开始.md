# 快速开始

这份文档面向两类使用方式：

- 想最快把整套系统跑起来并验证四端登录分流
- 想分别启动后端和各前端工程做联调

## 1. 最快启动方式

在项目根目录执行：

```bash
docker-compose up -d
```

启动完成后访问：

- 管理后台：`http://127.0.0.1:3000`
- 教师端：`http://127.0.0.1:3001`
- 学生端：`http://127.0.0.1:3002`
- 家长端：`http://127.0.0.1:3003`
- 后端 API：`http://127.0.0.1:8000`

当前 Docker Compose 默认启动：

- PostgreSQL
- Redis
- Django 后端
- 管理后台
- 教师端
- 学生端
- 家长端

## 2. 默认账号

- 管理后台：`admin / admin123`
- 教师端：`teacher / teacher123`
- 学生端：`13800000002 / student123`
- 家长端：`13800000003 / parent123`

## 3. 统一登录分流

当前管理后台登录页支持：

- 管理员登录后台
- 用户注册
- 忘记密码
- 邮箱验证码

并支持按角色自动分流：

- 教师登录后台登录页后自动跳转到教师端
- 学生登录后台登录页后自动跳转到学生端
- 家长登录后台登录页后自动跳转到家长端

## 4. 启用邮箱验证码

在根目录 `.env` 中配置后端 SMTP 参数，例如：

```bash
EMAIL_HOST=smtp.qq.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@example.com
EMAIL_HOST_PASSWORD=your-app-password
EMAIL_USE_TLS=True
DEFAULT_FROM_EMAIL=your-email@example.com
```

配置完成后重建后端：

```bash
docker-compose up -d --build backend
```

## 5. 本地启动后端

在 `04_后端开发/03_业务代码` 目录执行：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
python manage.py runserver 8000
```

## 6. 本地启动管理后台

在 `03_前端开发/01_管理后台` 目录执行：

```bash
npm install
npm run dev
```

默认访问地址通常为：`http://127.0.0.1:3000`

## 7. 本地启动三端客户端

三端 Taro H5 已固定本地开发端口：

- 教师端：`3001`
- 学生端：`3002`
- 家长端：`3003`

进入各自目录后执行：

```bash
npm install
npm run dev:h5
```

如果只需要验证是否可构建：

```bash
npm run build:h5
```

## 8. 常见场景

### 只想看完整演示

直接使用 Docker 启动，然后从 `3000` 登录页验证管理员后台和用户分流。

### 只改后端接口

本地启动后端即可，不必同时启动全部前端工程。

### 只调教师/学生/家长端

可以只启动对应前端工程，再复用本地或 Docker 后端 `8000` 端口。

## 9. 当前边界

- 管理后台当前只允许管理员进入。
- 非管理员统一从管理后台登录页分流到各自端。
- 三端已支持 Docker 访问和参数自动登录，但仍以教学演示和联调为主。
