# 快速开始

这份文档面向两类使用方式：

- 想尽快把系统跑起来并访问管理后台
- 想在本地分别启动后端和前端做开发联调

## 1. 最快启动方式

在项目根目录执行：

```bash
docker-compose up -d
```

启动完成后访问：

- 管理后台：`http://127.0.0.1:3000`
- 后端 API：`http://127.0.0.1:8000`

当前 Docker Compose 仅启动以下服务：

- PostgreSQL
- Redis
- Django 后端
- 管理后台

## 2. 默认账号

- 管理后台：`admin / admin123`
- 教师端：`teacher / teacher123`
- 学生端：`13800000002 / student123`
- 家长端：`13800000003 / parent123`

这些账号由后端初始化命令自动创建；即使你只先访问管理后台，也建议保留它们作为联调和接口验证账号。

## 3. 本地启动后端

在 `04_后端开发/03_业务代码` 目录执行：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py init_subjects
python manage.py init_admin
python manage.py init_demo_users
python manage.py runserver 8000
```

说明：

- 未配置 `DB_HOST`、`DB_NAME` 等环境变量时，后端默认使用项目内的 SQLite。
- 配置了数据库相关环境变量后，后端会自动切换到 PostgreSQL。
- 未配置 `REDIS_URL` 时，缓存会退回本地内存缓存。

## 4. 本地启动管理后台

在 `03_前端开发/01_管理后台` 目录执行：

```bash
npm install
npm run dev
```

默认访问地址通常为：`http://127.0.0.1:3000`

## 5. 本地启动三端客户端

教师端、学生端、家长端都是独立的 Taro 工程。进入各自目录后执行：

```bash
npm install
npm run dev:h5
```

对应目录：

- 教师端：`03_前端开发/02_教师端`
- 学生端：`03_前端开发/03_学生端`
- 家长端：`03_前端开发/04_家长端`

如果只需要验证是否可构建，可执行：

```bash
npm run build:h5
```

## 6. 常见场景

### 只想看系统演示

使用 Docker 启动即可，优先访问管理后台。

### 只改后端接口

本地启动后端即可，不必同时启动全部前端工程。

### 需要联调管理后台

本地启动后端和管理后台；如果后端用的是本地 `8000` 端口，管理后台开发模式可直接联调。

### 需要查看三端工程是否还可用

分别进入三端目录执行 `npm run build:h5` 或 `npm run dev:h5`。

## 7. 当前边界

- 管理后台是当前最完整的 Web 入口。
- 教师端、学生端、家长端当前是演示版客户端，具备源码和构建能力，但不在默认 Docker 部署范围。
- 课堂链路包含创建、开始、结束、录制、回放等接口，但目前仍以 Mock/演示链路为主。
